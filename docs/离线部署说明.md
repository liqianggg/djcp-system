# DJCP 等保测评管理系统 — 离线部署说明

**版本**: v1.1  
**适用场景**: 内网环境，无外网访问，无法执行 `npm install`

---

## 关于数据库

系统当前使用 **SQLite**（通过 better-sqlite3），这是最适合离线/内网部署的数据库：

| 对比项 | SQLite（当前） | MySQL | PostgreSQL |
|--------|:---:|:---:|:---:|
| 需要独立服务 | ❌ 不需要 | ✅ 需要 | ✅ 需要 |
| 网络依赖 | ❌ 无 | ✅ 需要 | ✅ 需要 |
| 配置文件 | 零配置 | 需配置 | 需配置 |
| 备份方式 | 复制 .db 文件 | 需 mysqldump | 需 pg_dump |
| 迁移难度 | 复制文件即可 | 导出导入 | 导出导入 |
| 适用场景 | 单机/内网 | 高并发 | 复杂查询 |

> **结论**：SQLite 是内网单机部署的最佳选择，无需额外安装数据库服务，无需网络连接。无需更换。

## 方式一：离线完整包部署（推荐）

### 步骤 1：解压

```bash
tar -xzf ~/Desktop/djcp-system-offline-v1.1.tar.gz -C ~/Desktop/djcp-system
cd ~/Desktop/djcp-system
```

### 步骤 2：确认平台兼容性

离线包中的 `better-sqlite3` 是 macOS ARM64 版本的编译二进制文件。

- **如果目标机器也是 macOS ARM64（M1/M2/M3/M4）**：无需额外操作
- **如果目标机器是 macOS x86_64（Intel）或 Linux**：需要重新编译原生模块（见下方"跨平台部署"）

### 步骤 3：直接启动

```bash
# 启动后端服务（Express 托管前端静态文件）
cd server
node src/index.js
```

服务启动后访问：`http://localhost:3001`

> 前端已预构建在 `client/dist/`，Express 自动托管，无需额外启动前端服务。

### 步骤 4：验证

浏览器打开 `http://localhost:3001`，使用默认账号 `sysadmin` / `admin123` 登录。

## 方式二：源码包部署（需要能访问 npm）

```bash
tar -xzf ~/Desktop/djcp-system-v1.1.tar.gz -C ~/Desktop/djcp-system
cd ~/Desktop/djcp-system

# 安装依赖（需要网络）
cd server && npm install
cd ../client && npm install && npm run build

# 启动
cd ../server && node src/index.js
```

## 跨平台部署

如果目标机器不是 macOS ARM64，需要重新编译原生模块：

### 在目标机器上操作

```bash
cd server

# 仅重新安装原生模块（better-sqlite3）
npm rebuild better-sqlite3

# 或者完整重新安装
rm -rf node_modules
npm install --production
```

> `npm rebuild` 会在本地重新编译原生 .node 文件，匹配当前平台。

### 如果目标机器也完全无法联网

**方案 A**：在开发机上为目标平台预编译
```bash
# 在开发机（可联网）上为目标平台安装
npm install --target_arch=x64 --target_platform=linux
```

**方案 B**：建立内网 npm 私有镜像
- 使用 [Verdaccio](https://verdaccio.org/) 搭建轻量级 npm 私有仓库
- 在一台可访问外网的机器上下载所有依赖，然后导入内网

**方案 C**：使用 pnpm offline store
```bash
# 在开发机上
pnpm install
tar -czf pnpm-store.tar.gz ~/.pnpm-store

# 复制到目标机器
tar -xzf pnpm-store.tar.gz -C ~/
cd server && pnpm install --offline
```

## 服务管理

### 后台运行（使用 nohup）

```bash
cd server
nohup node src/index.js > logs/server.log 2>&1 &
```

### 使用 systemd（Linux）

```ini
# /etc/systemd/system/djcp.service
[Unit]
Description=DJCP 等保测评管理系统
After=network.target

[Service]
Type=simple
User=youruser
WorkingDirectory=/path/to/djcp-system/server
ExecStart=/usr/bin/node src/index.js
Restart=on-failure

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable djcp --now
```

### 使用 launchd（macOS）

```xml
<!-- ~/Library/LaunchAgents/com.djcp.server.plist -->
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN"
  "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>Label</key>
    <string>com.djcp.server</string>
    <key>ProgramArguments</key>
    <array>
        <string>/usr/local/bin/node</string>
        <string>/path/to/djcp-system/server/src/index.js</string>
    </array>
    <key>WorkingDirectory</key>
    <string>/path/to/djcp-system/server</string>
    <key>RunAtLoad</key>
    <true/>
    <key>KeepAlive</key>
    <true/>
</dict>
</plist>
```

```bash
launchctl load ~/Library/LaunchAgents/com.djcp.server.plist
```

## 常见问题

### Q: 启动报错 "better_sqlite3.node: invalid ELF header"

平台不匹配。在目标机器上执行 `cd server && npm rebuild better-sqlite3`。

### Q: 如何备份数据？

复制 `server/djcp.db` 文件即可，这是完整的 SQLite 数据库。

```bash
cp server/djcp.db server/djcp.db.backup.$(date +%Y%m%d)
```

### Q: 如何迁移到新机器？

1. 在新机器上解压离线包
2. 确认原生模块兼容（或 rebuild）
3. 复制 `server/djcp.db` 到新机器的 `server/` 目录
4. 复制 `server/uploads/` 目录（如有上传文件）
5. 启动服务

### Q: 可以同时运行多个实例吗？

SQLite 支持多读单写，但建议只运行一个实例。如需多实例部署，考虑切换为 PostgreSQL。
