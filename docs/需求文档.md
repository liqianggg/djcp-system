# 网络安全等级保护测评全生命周期管理系统 — 需求文档

**版本**: 1.0  
**日期**: 2026-05-31  
**文档编号**: DJCP-REQ-1.0

---

## 1. 引言

### 1.1 编写目的

本文档定义"网络安全等级保护测评全生命周期管理系统"（以下简称 DJCP 系统）的功能需求与非功能需求，作为设计、开发和验收的依据。

### 1.2 项目背景

依据《中华人民共和国网络安全法》及《信息安全技术 网络安全等级保护基本要求》（GB/T 22239-2019），网络运营者须对信息系统开展等级保护定级、备案、整改、测评等全生命周期管理。本系统为组织提供统一的等保测评流程管理平台。

### 1.3 适用范围

适用于政府机关、企事业单位信息中心及安全管理部门对多个信息系统的等保测评全流程管理。

### 1.4 术语定义

| 术语 | 说明 |
|------|------|
| 等保 | 网络安全等级保护（等级保护） |
| 定级 | 根据 GB/T 22240-2020 确定信息系统安全保护等级（1-5 级） |
| 备案 | 向公安机关提交定级报告进行备案登记 |
| 差距分析 | 对比现状与等保标准要求的差距评估 |
| 整改 | 针对差距分析结果实施安全整改 |
| 测评 | 由测评机构对系统进行等级测评 |
| LDAP | 轻量目录访问协议，用于域控账户认证 |

## 2. 系统总体需求

### 2.1 系统角色

| 角色 | 标识 | 权限范围 |
|------|------|----------|
| 系统管理员 | system_admin | 全部权限，含系统配置及用户管理 |
| 安全管理员 | security_admin | 业务功能 + 用户管理 + 配置管理 |
| 安全审计员 | security_auditor | 只读查看 + 审计日志 |
| 操作员 | operator | 业务功能（无用户/权限管理） |
| 只读用户 | viewer | 仅查看 |

### 2.2 技术环境

| 层级 | 技术栈 |
|------|--------|
| 前端 | React 19 + Vite 8 + React Router 7 |
| 后端 | Node.js + Express 5 |
| 数据库 | SQLite (better-sqlite3) |
| 认证 | JWT + bcryptjs，支持 LDAP/AD 域控 |
| 图表 | Recharts |
| 图标 | Lucide React |
| PDF 导出 | PDFKit |

## 3. 功能需求

### 3.1 工作台 (Dashboard)

**需求编号**: REQ-001

- 展示统计概览：信息系统总数、已定级数、已备案数、整改中数
- 按安全等级分布展示饼图/柱状图
- 展示最近操作记录

### 3.2 信息系统管理

**需求编号**: REQ-002

- REQ-002-01: 新增信息系统（名称、编号、所属部门、系统类别 S1-S3/G1-G3、描述、安全等级）
- REQ-002-02: 编辑/删除信息系统
- REQ-002-03: 列表查看，支持按状态筛选
- REQ-002-04: 系统状态流转：draft → classified → filed → assessing → rectifying → completed

### 3.3 系统定级管理

**需求编号**: REQ-003

- REQ-003-01: 对信息系统进行定级（业务信息安全等级 1-5、服务范围、业务依赖描述、定级依据说明）
- REQ-003-02: 定级结果存入 classifications 表，同步更新系统安全等级
- REQ-003-03: 生成定级报告（HTML 在线预览 + PDF 下载）
- REQ-003-04: 报告包含 5 个章节：基本信息、定级结果、定级依据、定级矩阵、定级说明

### 3.4 备案管理

**需求编号**: REQ-004

- REQ-004-01: 新增备案记录（备案号、备案机关、备案日期、备案年份、状态）
- REQ-004-02: 备案状态：preparing → submitted → approved → rejected
- REQ-004-03: 上传备案证明图片（支持 PNG/JPG/PDF，多文件）
- REQ-004-04: 备案证明图片预览与下载
- REQ-004-05: 按年份筛选备案记录
- REQ-004-06: 备案批准后自动更新系统状态为 filed

### 3.5 差距分析

**需求编号**: REQ-005

- REQ-005-01: 创建差距分析记录（关联信息系统、分析日期）
- REQ-005-02: 添加差距项（需求类别、需求编号、需求描述、期望值、实际值、合规状态、风险等级）
- REQ-005-03: 导入 Excel 文件一键识别差距项
- REQ-005-04: 计算合规率与总体评分

### 3.6 整改管理

**需求编号**: REQ-006

- REQ-006-01: 创建整改任务（关联信息系统与差距项、标题、描述、责任人、优先级、计划日期）
- REQ-006-02: 整改状态：pending → in_progress → completed → verified
- REQ-006-03: 上传整改证明截图（支持 PNG/JPG）
- REQ-006-04: 整改证明图片在线查看

### 3.7 测评管理

**需求编号**: REQ-007

- REQ-007-01: 创建测评记录（测评机构、类型 initial/reassessment/annual、日期、报告编号）
- REQ-007-02: 添加测评项（类别、控制点编号、描述、得分、满分、结果 符合/部分符合/不符合/不适用）
- REQ-007-03: 测评结论：pass / fail / conditional_pass

### 3.8 文档管理

**需求编号**: REQ-008

- REQ-008-01: 上传文档（关联信息系统、标题、类型、版本号、描述）
- REQ-008-02: 文档类型：policy / procedure / record / report / evidence / other
- REQ-008-03: 文档下载
- REQ-008-04: 文档编辑与删除

### 3.9 用户与权限管理

**需求编号**: REQ-009

- REQ-009-01: 用户 CRUD（本地账户 + LDAP 账户）
- REQ-009-02: 用户状态：active / disabled
- REQ-009-03: 重置密码
- REQ-009-04: 基于角色的权限控制（RBAC），细粒度到操作级（如 `filing:create`）
- REQ-009-05: 权限管理界面，支持为角色分配权限码

### 3.10 审计日志

**需求编号**: REQ-010

- REQ-010-01: 记录所有操作（操作人、操作类型、目标类型、目标 ID、描述、结果）
- REQ-010-02: 按时间范围/操作类型/用户筛选
- REQ-010-03: 导出为 CSV

### 3.11 系统管理

**需求编号**: REQ-011

- REQ-011-01: LDAP/AD 域控配置（服务器地址、端口、Base DN、域名、管理员账号密码）
- REQ-011-02: LDAP 连接验证测试功能
- REQ-011-03: 文件上传路径配置
- REQ-011-04: 配置项持久化存储

### 3.12 认证与安全

**需求编号**: REQ-012

- REQ-012-01: 本地密码登录（bcrypt 哈希）
- REQ-012-02: LDAP/AD 域控登录
- REQ-012-03: JWT 令牌认证（24 小时有效期）
- REQ-012-04: 登录速率限制（每 15 分钟最多 10 次失败尝试）
- REQ-012-05: 全局 API 速率限制（每 15 分钟 500 次请求）
- REQ-012-06: 安全响应头（helmet）
- REQ-012-07: CORS 限制受信任来源

## 4. 非功能需求

### 4.1 性能

- 页面首次加载 ≤ 3 秒
- API 响应 ≤ 500ms（P95）
- 支持并发用户数 ≥ 50

### 4.2 安全性

- 密码哈希存储（bcrypt，cost factor 10）
- JWT 签名验证所有 API 请求
- 操作级权限校验
- 审计日志不可篡改

### 4.3 可用性

- 仿 Apple 风格毛玻璃 UI 设计
- 侧边栏导航，按业务功能/测评流程/资源管理/系统管理分组
- 响应式布局，适配不同浏览器分辨率
- 中文界面

### 4.4 可维护性

- 前后端分离架构
- 数据持久化 SQLite，零配置部署
- 清晰的文件组织结构

## 5. 业务流程

```
信息系统登记 → 系统定级 → 定级报告 → 备案登记 → 
差距分析 → 整改实施 → 等级测评 → 完成
```

每个阶段均产生对应记录，并可回溯审计。

## 6. 附录

### 6.1 安全等级定义

| 等级 | 名称 | 说明 |
|------|------|------|
| 1 | 自主保护级 | 一般损害 |
| 2 | 指导保护级 | 严重损害 |
| 3 | 监督保护级 | 特别严重损害 |
| 4 | 强制保护级 | — |
| 5 | 专控保护级 | — |

- S1: 业务信息安全-一般
- S2: 业务信息安全-严重
- S3: 业务信息安全-特别严重
- G1: 系统服务安全-一般
- G2: 系统服务安全-严重
- G3: 系统服务安全-特别严重

- GB/T 22239-2019 网络安全等级保护基本要求
