## 1. 概述

### 1.1 系统架构

```
┌──────────────────────────────────────────────────────────────┐
│                        浏览器 (Browser)                        │
│  ┌──────────────────────────────────────────────────────────┐ │
│  │              React SPA (Vite 8 构建)                      │ │
│  │  App.jsx ─── Router ─── 12 个页面组件                     │ │
│  │     │                    (Dashboard/Systems/…)            │ │
│  │     └── api.js ── 封装 fetch 请求                          │ │
│  └──────────────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────────────┘
                              │ HTTP (REST API)
                              ▼
┌──────────────────────────────────────────────────────────────┐
│                    Express 5 Server (Node.js)                  │
│                                                                │
│  index.js ─── 中间件 (helmet/cors/rate-limit)                  │
│     │                                                          │
│     └── routes.js ─── 50+ REST API 端点                       │
│              │                                                 │
│              ├── 认证 (JWT + bcrypt + LDAP)                    │
│              ├── RBAC 中间件 (requirePermission)                │
│              ├── 文件上传 (multer)                              │
│              └── 审计日志 (auditLog)                            │
│                    │                                           │
│                    └── database.js ── better-sqlite3 ── SQLite │
└──────────────────────────────────────────────────────────────┘
```

### 1.2 技术选型理由

| 技术 | 理由 |
|------|------|
| SQLite | 零配置部署，适合单机/内网场景，无需独立数据库服务 |
| Express 5 | 成熟稳定的 Node.js Web 框架 |
| React 19 + Vite 8 | 现代化前端工具链，快速开发构建 |
| JWT | 无状态认证，适合前后端分离架构 |
| PDFKit | 纯 JavaScript PDF 生成，支持中文字体 |
| better-sqlite3 | 同步 API 简化代码，性能优异 |

### 3.4 权限码体系

所有业务 API 均通过 `requirePermission(code)` 中间件进行权限校验。权限码命名规范：`{module}:{action}`

完整权限码列表：
```
dashboard:view
system:view, system:create, system:edit, system:delete
classification:view, classification:create, classification:report
filing:view, filing:create, filing:edit, filing:upload_evidence, filing:delete_evidence
gap:view, gap:create, gap:import
rectification:view, rectification:create, rectification:edit, rectification:status
assessment:view, assessment:create
document:view, document:upload, document:edit, document:delete, document:download
user:view, user:create, user:edit, user:delete, user:reset_password
permission:view, permission:manage
audit:view, audit:export
settings:view
```

### 3.5 认证流程

```
用户登录
   ├── login_type === 'ldap'
   │     └── ldapAuthenticate(username, password, ldapSettings)
   │           └── LDAP bind → 成功/失败
   └── login_type === 'local'
         └── bcrypt.compareSync(password, user.password)
               └── 匹配/不匹配
   ↓
生成 JWT (payload: {id, username, role}, expires: 24h)
   ↓
存入 sessions 表
   ↓
返回 {success, token, user(含 permissions 列表)}
```

## 5. 安全设计

### 5.1 认证安全

- 密码: bcrypt (cost factor 10) 哈希存储
- LDAP: 不存储域控密码，仅实时 bind 验证
- JWT: HMAC-SHA256 签名，24h 过期
- 登录限流: 每 IP 每 15 分钟 10 次失败即锁定

### 5.2 授权安全

- 每个 API 端点通过 `requirePermission(code)` 中间件校验
- 前端路由也进行权限检查（`hasPerm` 函数），无权限跳转
- 侧边栏菜单项按权限过滤显示

### 5.3 传输安全

- CORS 限制为前端地址 `http://localhost:5173`
- helmet 中间件设置安全响应头
- 全局速率限制 500 次/15 分钟

### 5.4 审计安全

- 所有操作写入 audit_logs（含失败登录尝试）
- 审计日志表不可通过 API 删除
