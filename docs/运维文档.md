## 2. 部署指南

### 2.1 快速部署

```bash
# 1. 克隆或解压项目到目标目录
cd /opt/djcp-system

# 2. 安装后端依赖
cd server
npm install

# 3. 安装前端依赖并构建
cd ../client
npm install
npm run build

# 4. 启动服务
cd ../server
npm start
```

服务启动后：
- API 服务: `http://localhost:3001`
- 前端页面: `http://localhost:3001` (静态文件由 Express 托管)

### 2.2 开发模式

```bash
# 终端1 - 后端
cd server
npm run dev     # 带 --watch 自动重启

# 终端2 - 前端
cd client
npm run dev     # Vite 热更新, 默认 http://localhost:5173
```

开发模式下前端通过 Vite 代理转发 API 请求到 `localhost:3001`。

### 2.3 环境变量

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `PORT` | 3001 | 服务端口 |
| `JWT_SECRET` | djcp_jwt_secret_key_2026 | JWT 签名密钥（生产环境务必修改） |
| `CORS_ORIGIN` | http://localhost:5173 | 允许的跨域来源 |

```bash
# 生产环境启动示例
PORT=8080 JWT_SECRET=your_secure_random_string npm start
```

## 3. 初始配置

### 3.1 默认账户

系统首次启动时自动创建以下账户（密码均为 `admin123`）：

| 用户名 | 角色 | 姓名 |
|--------|------|------|
| sysadmin | 系统管理员 | 系统管理员 |
| secadmin | 安全管理员 | 安全管理员 |
| auditor | 安全审计员 | 安全审计员 |
| operator | 操作员 | 操作员 |
| viewer | 只读用户 | 只读用户 |

> **首次登录后请立即修改密码！**

### 3.2 LDAP/AD 域控配置（可选）

1. 使用 sysadmin 登录
2. 进入"系统管理"页面
3. 填写以下配置：
   - 启用域控登录: true
   - LDAP 服务器地址: 如 `192.168.1.10`
   - LDAP 端口: 默认 389
   - Base DN: 如 `dc=company,dc=com`
   - AD 域名: 如 `company.com`
   - 管理员账号/密码
4. 点击"测试连接"验证
5. 保存配置

配置 LDAP 后，在用户管理中添加 `login_type=ldap` 的用户即可使用域控账户登录。

### 3.3 文件上传路径配置

1. 进入"系统管理"页面
2. 修改 `upload_path` 配置项（默认 `uploads`，相对于 server 目录）
3. 确保目标目录存在且有写入权限

## 4. 日常运维

### 4.1 启动与停止

```bash
# 启动
cd /opt/djcp-system/server
nohup node src/index.js > /var/log/djcp.log 2>&1 &

# 停止
kill $(lsof -t -i:3001)
```

### 4.2 使用 systemd 管理（Linux 推荐）

创建 `/etc/systemd/system/djcp.service`：

```ini
[Unit]
Description=DJCP 等保测评管理系统
After=network.target

[Service]
Type=simple
User=djcp
WorkingDirectory=/opt/djcp-system/server
ExecStart=/usr/bin/node src/index.js
Restart=on-failure
RestartSec=5
Environment=NODE_ENV=production
Environment=PORT=3001
Environment=JWT_SECRET=your_secure_secret_here

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable djcp
sudo systemctl start djcp
sudo systemctl status djcp
```

### 4.3 日志

- 应用日志输出到标准输出，可通过重定向或 systemd journal 收集
- 操作审计日志存储在数据库 `audit_logs` 表中，可通过系统界面查看和导出

### 4.4 数据备份

```bash
# 备份 SQLite 数据库
cp /opt/djcp-system/server/djcp.db /backup/djcp_$(date +%Y%m%d).db

# 备份上传文件
tar -czf /backup/djcp_uploads_$(date +%Y%m%d).tar.gz /opt/djcp-system/server/uploads/

# 建议每日定时备份
# crontab 示例:
# 0 2 * * * cp /opt/djcp-system/server/djcp.db /backup/djcp_$(date +\%Y\%m\%d).db
```

### 4.5 数据恢复

```bash
# 停止服务
sudo systemctl stop djcp

# 恢复数据库
cp /backup/djcp_20260531.db /opt/djcp-system/server/djcp.db

# 恢复上传文件
tar -xzf /backup/djcp_uploads_20260531.tar.gz -C /

# 启动服务
sudo systemctl start djcp
```

## 6. 故障排查

### 6.1 服务无法启动

**问题**: `npm start` 报错
```bash
# 检查 Node.js 版本
node -v  # 需要 ≥ 18

# 重新安装依赖
cd server && rm -rf node_modules && npm install

# 检查数据库文件是否有写权限
ls -la server/djcp.db
chmod 644 server/djcp.db

# 检查端口是否被占用
lsof -i :3001
```

### 6.2 前端页面空白

```bash
# 重新构建前端
cd client && npm run build

# 确认 dist 目录存在
ls -la client/dist/index.html
```

### 6.3 PDF 报告中文乱码

```bash
# 确认中文字体存在
ls -la '/Library/Fonts/Arial Unicode.ttf'

# macOS: 系统自带
# Linux: 安装中文字体
#   sudo apt-get install fonts-wqy-zenhei
#   或 sudo yum install wqy-zenhei-fonts
#   然后修改 server/src/routes.js 中的 FONT_PATH

# Windows:
#   FONT_PATH = 'C:\\Windows\\Fonts\\msyh.ttc'
```

### 6.4 文件上传失败

```bash
# 检查 uploads 目录权限
mkdir -p server/uploads
chmod 755 server/uploads

# 检查磁盘空间
df -h
```

### 6.5 登录失败

- 确认用户状态为 `active`：`SELECT * FROM users WHERE username='xxx';`
- 如密码遗忘，通过 SQL 重置（见 5.2 节）
- LDAP 用户登录失败：检查"系统管理"中 LDAP 配置是否正确，点击"测试连接"验证

## 8. 安全运维建议

1. **生产环境必须修改 `JWT_SECRET`**，使用随机 64 位字符串
2. **定期更改管理员密码**（每 90 天）
3. **启用 HTTPS**（通过 Nginx 反向代理 + Let's Encrypt 证书）
4. **配置防火墙**（仅开放 443 端口）
5. **定期备份数据库**（建议每日）
6. **定期审查审计日志**，关注异常登录
7. **保持 Node.js 版本更新**

### 8.1 Nginx 反向代理配置（可选）

```nginx
server {
    listen 443 ssl;
    server_name djcp.example.com;

    ssl_certificate /etc/letsencrypt/live/djcp.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/djcp.example.com/privkey.pem;

    location / {
        proxy_pass http://127.0.0.1:3001;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```
