## 1. 系统概述

本系统用于网络安全等级保护（等保）测评的全生命周期管理，覆盖信息系统定级、备案、差距分析、整改、测评的完整流程。系统采用三权分立权限模型，确保安全管理、业务操作、审计监督相互制衡。

### 核心业务流程

```
信息系统登记 → 系统定级 → 生成定级报告 → 备案登记 → 
差距分析 → 整改实施 → 等级测评 → 完成
```

### 系统角色

| 角色 | 职责 | 典型操作 |
|------|------|----------|
| 系统管理员 | 系统配置、用户管理、全部权限 | 创建用户、配置 LDAP、分配权限 |
| 安全管理员 | 业务管理 + 用户管理 | 定级、备案、整改、测评、管理用户 |
| 安全审计员 | 审计监督 | 查看操作日志、导出审计报告 |
| 操作员 | 日常业务操作 | 创建系统、差距分析、整改执行 |
| 只读用户 | 查阅信息 | 浏览各模块数据 |

## 2. 快速入门

### 2.1 登录系统

1. 打开浏览器访问系统地址（默认 `http://localhost:3001`）
2. 输入用户名和密码
3. **本地登录**：直接输入账号密码，点击「登录」
4. **域控登录**：勾选「域控登录 (LDAP/AD)」复选框，输入域控账号密码，点击「登录」

### 2.2 默认账号

| 用户名 | 密码 | 角色 | 说明 |
|--------|------|------|------|
| `sysadmin` | `admin123` | 系统管理员 | 全部权限 |
| `secadmin` | `admin123` | 安全管理员 | 业务 + 用户管理 |
| `auditor` | `admin123` | 安全审计员 | 只读 + 审计 |
| `operator` | `admin123` | 操作员 | 业务功能 |
| `viewer` | `admin123` | 只读用户 | 仅查看 |

> ⚠️ **安全提示**：首次登录后应立即修改默认密码。

### 2.3 界面布局

登录后界面分为三个区域：

- **左侧边栏**：导航菜单，按功能分组（业务功能、测评流程、资源管理、系统管理）
- **顶部标题栏**：当前页面标题 + 主要操作按钮
- **主内容区**：搜索筛选栏 + 数据表格 + 弹窗表单

## 3. 业务功能

### 3.1 工作台

**路径**: 侧边栏 → 工作台

工作台为登录后的首页，展示系统整体概况：

| 功能 | 说明 |
|------|------|

### 3.2 信息系统管理

**路径**: 侧边栏 → 业务功能 → 信息系统

#### 3.2.1 新建信息系统

1. 点击右上角「新建系统」按钮
2. 填写系统信息：
   - **名称** \*（必填）：信息系统的正式名称
   - **编号**：系统唯一编号，用于标识
   - **所属部门**：该系统的归属部门
   - **类别**：S1/S2/S3（业务信息安全）或 G1/G2/G3（系统服务安全）
   - **安全等级**：1-5 级
   - **描述**：系统功能简介
3. 点击「保存」

#### 3.2.2 编辑 / 删除

- 在列表行右侧点击 ✏️ 图标进入编辑
- 点击 🗑️ 图标删除（会同时删除关联的定级、备案、测评等记录）

#### 3.2.3 搜索与筛选

- 在搜索框输入系统名称、编号或部门关键字进行搜索
- 使用状态下拉框筛选特定生命周期阶段的系统

#### 3.2.4 生命周期状态

| 状态 | 含义 |
|------|------|
| 草稿 | 新建系统，尚未定级 |
| 已定级 | 已完成安全等级确定 |
| 已备案 | 已向公安机关备案 |
| 测评中 | 正在进行等级测评 |
| 整改中 | 正在根据差距分析结果整改 |
| 已完成 | 全流程完成 |

---

### 3.3 系统定级

**路径**: 侧边栏 → 业务功能 → 系统定级

#### 3.3.1 新建定级

1. 点击「新建定级」按钮
2. 选择目标信息系统
3. 填写定级信息：
   - **业务信息安全等级**（1-5 级）
   - **服务范围**
   - **业务依赖描述**
   - **定级依据说明**
4. 点击「保存」，定级结果将同步更新系统的安全等级

#### 3.3.2 生成定级报告

- 在定级记录行点击「生成报告」
- 支持 **HTML 在线预览** 和 **PDF 下载** 两种格式
- 报告包含：基本信息、定级结果、定级依据、定级矩阵、定级说明

### 3.4 备案管理

**路径**: 侧边栏 → 测评流程 → 备案管理

#### 3.4.1 新建备案

1. 点击「新建备案」按钮
2. 选择信息系统
3. 填写备案信息：
   - **备案编号** \*（必填）
   - **备案机关**
   - **备案日期** / **审批日期**
   - **备案年份**
   - **状态**：准备中 → 已提交 → 已审批 → 已驳回
4. 点击「创建」

#### 3.4.2 上传备案证明

- 进入编辑模式后，点击「上传证明」按钮
- 支持 PNG / JPG / PDF 格式
- 上传后可点击图片在线预览
- 点击 ✕ 可删除单个证明

#### 3.4.3 年份与状态筛选

使用工具栏的年份下拉框和状态下拉框进行组合筛选。

### 3.5 差距分析

**路径**: 侧边栏 → 测评流程 → 差距分析

#### 3.5.1 新建差距分析

1. 点击「新建差距分析」按钮
2. 选择信息系统和分析日期
3. 逐项评估各安全要求的符合情况：
   - 对比 **期望值** 与 **实际值**
   - 判定合规状态（是/否）
   - 设定风险等级（高/中/低）
4. 系统自动计算合规率与总体评分

#### 3.5.2 Excel 导入

1. 点击「导入 Excel」按钮
2. 选择包含差距项的 Excel 文件
3. 系统自动解析并导入差距项

#### 3.5.3 状态说明

| 状态 | 含义 |
|------|------|
| 草稿 | 分析中，尚未完成 |
| 进行中 | 正在评估 |
| 已完成 | 差距分析已完成 |

### 3.6 整改管理

**路径**: 侧边栏 → 测评流程 → 整改管理

#### 3.6.1 创建整改任务

1. 点击「新建整改」按钮
2. 选择信息系统
3. 填写任务信息：
   - **标题** \*（必填）
   - **描述**
   - **责任人**
   - **优先级**：紧急 → 高 → 中 → 低
   - **计划开始 / 完成日期**
4. 点击「保存」

#### 3.6.2 状态跟踪

| 状态 | 含义 | 操作 |
|------|------|------|
| 待处理 | 任务已创建，尚未开始 | 开始执行 |
| 进行中 | 正在执行整改 | 标记完成 |
| 已完成 | 整改工作已完成 | 验证确认 |
| 已验证 | 经审核确认整改有效 | — |

#### 3.6.3 上传整改证明

- 在整改记录中可上传整改前后的截图作为证明
- 支持 PNG / JPG 格式
- 可在线查看上传的证明图片

### 3.7 测评管理

**路径**: 侧边栏 → 测评流程 → 测评管理

#### 3.7.1 新建测评

1. 点击「新建测评」按钮
2. 选择信息系统
3. 选择 **测评机构**（从机构库下拉选取，确保与测评机构管理模块关联）
4. 填写：
   - **测评类型**：初次测评 / 复评 / 年度测评
   - **测评日期**
   - **报告编号**
   - **状态**：计划中 → 进行中 → 已完成
5. 进入逐项评分界面，对等保控制点逐项打分

#### 3.7.2 评分标准

| 结果 | 得分范围 | 说明 |
|------|----------|------|
| 符合 | 满分 | 完全满足等保要求 |
| 部分符合 | 1 ~ 满分-1 | 大部分满足，存在小缺陷 |
| 不符合 | 0 | 不满足等保要求 |
| 不适用 | 0 | 该项目不适用于该信息系统 |

#### 3.7.3 测评结论

| 结论 | 含义 |
|------|------|
| 通过 | 测评合格 |
| 有条件通过 | 基本合格，存在需整改项 |
| 未通过 | 不合格，需全面整改 |

### 3.8 测评机构管理

**路径**: 侧边栏 → 资源管理 → 测评机构管理

#### 3.8.1 新增机构

1. 点击搜索框旁「新增机构」按钮
2. 填写机构信息：
   - **机构名称** \*（必填）：测评机构的正式名称
   - **资质等级**：国家级 / 省级 / 市级 / 其他
   - **资质编号**
   - **资质有效期**
   - **联系电话**
   - **机构邮箱**
   - **地址**
3. 填写对接人信息：
   - 对接人姓名、电话、邮箱
4. 填写备注（可选）
5. 点击「保存」

#### 3.8.2 编辑 / 删除机构

- 在列表行右侧点击 ✏️ 图标进入编辑
- 点击 🗑️ 图标删除（同时删除该机构下所有进场记录）

#### 3.8.3 查看 / 管理进场记录

1. 点击机构所在行展开详情

   - 甲方对接人员（从系统用户中选择）
   - 备注
4. 保存后记录出现在下方表格中

#### 3.8.4 搜索

在搜索框中输入机构名称或对接人姓名进行筛选。

## 5. 权限体系

### 5.1 三权分立原则

系统遵循等级保护标准要求的"三权分立"原则：

| 角色 | 相互制约 | 不可兼任 |
|------|----------|----------|
| 系统管理员 | 负责系统配置和用户管理 | 不可兼任审计 |
| 安全管理员 | 负责业务操作和安全管理 | 不可兼任审计 |
| 安全审计员 | 负责审计日志审查 | 不可兼任其他 |

### 5.2 权限验证

- **前端**：按钮和菜单项根据权限显示/隐藏
- **后端**：每个 API 端点均校验操作权限
- **双重保障**：即使绕过前端界面，API 层面也无法越权操作

### 5.3 域控用户默认权限

通过域控认证自动创建的用户，默认分配 `viewer`（只读用户）角色。系统管理员可在用户管理中为其调整角色。

## 6. 常见问题

### Q1: 忘记密码怎么办？

联系系统管理员在「用户管理」中使用「重置密码」功能。

### Q2: 域控登录提示"域控登录未启用，请联系系统管理员"？

说明系统未开启 LDAP 认证。请联系系统管理员在「系统设置」中启用并配置域控连接。

### Q3: 域控用户登录成功但看不到操作按钮？

域控自动创建的用户默认为「只读用户」角色。请联系系统管理员在「用户管理」中调整角色。

### Q4: 备案状态为"已审批"后系统状态未更新？

备案审批通过后系统会自动更新对应信息系统状态为"已备案"。如未更新，检查备案记录是否正确关联了信息系统。

### Q5: 删除信息系统后关联数据是否保留？

删除信息系统会级联删除其下的定级记录、备案记录、测评记录等。删除前请确认。

### Q6: 如何批量导入差距分析数据？

在差距分析页面点击「导入 Excel」，选择包含差距项的 Excel 文件。系统会自动解析并导入。

### Q7: 支持哪些文件格式上传？

- 备案证明：PNG / JPG / PDF
- 整改证明：PNG / JPG
- 文档管理：所有常见文件格式
- 差距分析导入：Excel (.xlsx / .xls)

### Q8: 测评机构与测评管理的关系？
